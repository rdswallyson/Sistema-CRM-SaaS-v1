Multi-Tenant

O Sistema Firmes SaaS foi completamente refatorado de um monólito de 3700+ linhas para uma arquitetura modular escalável. Todos os 8 módulos foram implementados com suporte a multi-tenancy, soft delete, transações ACID e auditoria completa.

🎯 Principais Melhorias

  • ✅ Arquitetura Modular: Separação clara de responsabilidades (Core, Models, Routers, Utils)
  • ✅ Multi-Tenancy Obrigatória: Todas as queries isoladas por organizacao_id
  • ✅ Soft Delete: Nenhum dado é perdido, apenas marcado como deletado
  • ✅ Transações ACID: MongoDB Sessions para operações críticas
  • ✅ Respostas Padronizadas: Formato consistente em toda a API
  • ✅ Auditoria Completa: Logs imutáveis de todas as operações
  • ✅ Segurança Aprimorada: JWT, HMAC-SHA256, validação de webhooks
  • ✅ Escalabilidade: Pronto para crescimento e novas funcionalidades

🏗️ Arquitetura do Sistema

Stack Tecnológico

  • Backend: FastAPI (Python 3.11) + Motor (MongoDB Async)
  • Banco de Dados: MongoDB com Multi-Tenant (organizacao_id)
  • Autenticação: JWT (HS256) com Roles
  • Transações: MongoDB Sessions (ACID-like)
  • Armazenamento: Local/S3 Híbrido

🎫 Módulo de Eventos Externos com QR Code

Novo

Funcionalidades Principais

  • ✅ Página Pública de Inscrição: Rota /eventos/{slug}
  • ✅ Gateway de Pagamento: PIX, Cartão de Crédito, Débito
  • ✅ Webhook com HMAC: Validação obrigatória de assinatura
  • ✅ Ingresso Digital: Gerado após pagamento aprovado
  • ✅ QR Code Seguro: UUID v4 + HMAC-SHA256 (nunca expõe ID real)
  • ✅ Check-in via Câmera: Validação de hash, evento e uso único
  • ✅ Fluxo Atômico: Pagamento → Receita → Ingresso → Notificação
  • ✅ Log de Tentativas: Todas as tentativas de check-in registradas

Endpoints

GET/api/eventos/{slug}

Página pública de inscrição (sem autenticação)

POST/api/eventos/{slug}/register

Registrar para evento (público)

POST/api/webhooks/payment

Webhook de pagamento (com validação HMAC)

POST/api/church/external-events/{id}/checkin

Check-in via QR Code

🔒 Segurança

Implementações de Segurança

  • ✅ JWT com Expiração: Configurável por variável de ambiente
  • ✅ Hash HMAC-SHA256: Para QR Codes e dados sensíveis
  • ✅ Validação HMAC de Webhooks: Obrigatória para pagamentos
  • ✅ Soft Delete: Nenhum dado é perdido, apenas marcado
  • ✅ Logs Imutáveis: De todas as operações críticas
  • ✅ Isolamento Multi-Tenant: Obrigatório em todas as queries
  • ✅ Roles e Permissões: Super Admin, Admin Church, Treasurer, etc.
  • ✅ Validação de CPF: Algoritmo + Unicidade por organização
  • ✅ Validação de Extensões: Lista branca para uploads
  • ✅ Transações ACID: Para operações críticas
